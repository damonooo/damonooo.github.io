0. Canvas

0.1 什么是Canvas

Canvas标签是HTML5中新增的元素，它提供了一系列API供JavaScript脚本调用，用来绘制图形、制作照片，创建动画，甚至可以进行实时视频处理或渲染。

0.2 基本原理

Canvas提供的API种类相当的繁多，这里就不一一详细列举了，先来大体说一下Canvas绘图、动画的一些基本原理，之后的具体使用我们在实例中边学边用。

我们先来想象一下画画时我们需要用到的东西：纸、笔、手。

一张纸让我们可以在它上面绘画，一支笔让我们可以在纸上留下痕迹，一只手让我们可以操控笔在纸的具体位置留下具体的图案。

Canvas的原理也是如此，它通过<canvas></canvas>标签配合width和height属性在html页面上设定一片区域作为canvas绘画的图纸，通过canvas.getContext()方法来生成一支笔，并通过JavaScript调用它提供的API来操控笔的位置以及画出来的图案。

动画又是如何实现的呢？?

我们平时看的动漫、游戏等，都是利用了人眼的视觉暂留现象，当多张图片连续快速播放时，遍会产生图片中的内容变成连续运动的动画的错觉。

玩游戏的同学肯定都听说过帧数这个概念，平时我们界定一个游戏画面卡不卡，通常都是用这个单位，每秒传输帧数（FPS）。这里的1帧就可以理解为一张图片，60帧/秒的含义就是，一秒钟之内会连续播放60张图片（渲染60次）。

而Canvas渲染图像的方式也是很简单粗暴，在渲染每一帧的时候，Canvas需要将指定区域的内容清空掉，经过计算位置之后，重新绘制这个区域的整个图像。

重绘的时机怎么控制？

三种方法，setInterval、setTimeout、requestAnimationFrame（推荐）。

2.1 背景

先来介绍一下游戏主要以用的雪碧图，如下：

背景作为游戏中的一个重要元素相信大家都很熟悉，在这个小游戏中，主要背景为恐龙脚下的地面，正是由于地面不断的向后移动，使得我们的主角小恐龙，看起来像是在向前奔跑一样。

而这种看似无限的滚动是如何实现的呢

其实是在canvas中同时绘制了两块地面元素，初始时一块在画面中，另一块在画面之外。动画开始时，第一块地面开始不断变化x坐标（向后移动），同时第二块地面也跟第一块地面一起移动，进入画面中，当地一块地面元素的x坐标完全移出画布中时，将他的x坐标改为画布最右点（画布之外）。这种享元模式便实现了地面的无限滚动。

这一节算是基础，下面我们通过代码来详细讲解实现一下。

首先我们在代码中创建一个HorizonLine类，用来代表我们游戏中的地面。

/**
 * canvas 地面将绘制到此画布上
 * spritePos 地面在雪碧图中的坐标
 */
function HorizonLine(canvas,spritePos) {
  this.spritePos = spritePos;                   // 地面元素在雪碧图中的坐标
  this.canvas = canvas;                         // Canvas画布对象
  this.ctx = canvas.getContext("2d");           // 创建画笔（上下文）
  this.dimensions = HorizonLine.dimensions;     // 地面元素在canvas中的形状位置   
  this.xPos = [];                               // 地面在画布中的x坐标
  this.yPos = 0;                                // 地面在画布中的y坐标

  this.setSourceDimesions();                    
  this.draw();
}

// 地面元素在canvas画布中的位置、大小常量
HorizonLine.dimensions = {
  WIDTH: 600,    // 宽600
  HEIGHT: 12,    // 高12像素
  YPOS: 127      // 在canvas中的位置
};

然后我们来写他的原型方法。主要的操作都在此处：

HorizonLine.prototype = {
  //地面在画布上的位置
  setSourceDimesions:function() {
    this.xPos = [0,this.dimensions.WIDTH];//0,600
    this.yPos = this.dimensions.YPOS;
  },
  // 绘制两块地面元素
  draw:function() {
    //使用9参数的drawImage方法
    this.ctx.drawImage(imgSprite,
    this.spritePos.x, this.spritePos.y,
    this.dimensions.WIDTH, this.dimensions.HEIGHT,
    this.xPos[0],this.yPos, // 根据x坐标实时绘制
    this.dimensions.WIDTH,this.dimensions.HEIGHT);

    this.ctx.drawImage(imgSprite,
    this.spritePos.x, this.spritePos.y,
    this.dimensions.WIDTH, this.dimensions.HEIGHT,
    this.xPos[1],this.yPos,
    this.dimensions.WIDTH,this.dimensions.HEIGHT);
  },

  // 更新两块地面元素的x坐标
  updateXPos:function(pos,increment) {
    var line1 = pos,
    line2 = pos === 0 ? 1 : 0;

    this.xPos[line1] -= increment;
    this.xPos[line2] = this.xPos[line1] + this.dimensions.WIDTH;

    //若第一段地面完全移出canvas外
    if(this.xPos[line1] <= -this.dimensions.WIDTH) {
      //则将其移动至canvas外右侧
      this.xPos[line1] += this.dimensions.WIDTH * 2;
      //同时将第二段地面移动至canvas内
      this.xPos[line2] = this.xPos[line1] - this.dimensions.WIDTH;
    }
  },

  // 重绘函数
  update:function(deltaTime,speed) {
    // 计算x坐标变化量的小公式。每帧移动3像素，每秒60帧，时间间隔
    var increment = Math.floor(speed * FPS * (deltaTime / 1000)); 

    if(this.xPos[0] <= 0) {//交换地面一和二
      this.updateXPos(0, increment);
    } else {
      this.updateXPos(1, increment);
    }
    this.draw();
  },
  reset:function() {
    this.xPos[0] = 0;
    this.xPos[1] = this.dimensions.WIDTH;
  }
};

2.3 小恐龙

下面我们来绘制小恐龙，基本可以确定小恐龙的几个状态，跑步中、跳跃中、游戏失败中，对应这三种状态会有不同的渲染逻辑在里面。

我们先从最基本的构造函数开始写起。

function Trex(canvas,spritePos){
    this.canvas = canvas;
    this.ctx = canvas.getContext('2d');
    this.spritePos = spritePos;     //在雪碧图中的位置
    this.xPos = 0;                  //在画布中的x坐标
    this.yPos = 0;                  //在画布中的y坐标
    this.groundYPos = 0;            //初始化地面的高度
    this.currentFrame = 0;          //初始化动画帧
    this.currentAnimFrames = [];    //记录当前状态的动画帧
    this.animStartTime = 0;         //动画开始的时间
    this.timer = 0; //计时器
    this.msPerFrame = 1000 / FPS;   //默认帧率
    this.config = Trex.config;  /   /拷贝一个配置的副本方便以后使用
    this.jumpVelocity = 0;          //跳跃的初始速度

    this.status = Trex.status.WAITING;  //初始化默认状态为待机状态

    //为各种状态建立标识
    this.jumping = false;           //角色是否处于跳跃中
    this.reachedMinHeight = false;  //是否到达最小跳跃高度
    this.jumpCount = 0;             //跳跃次数

    this.init();
}

2.4 操作控制

其实控制操作这种实现，大家应该已经猜到了，就是通过addEventListener来监听键盘的keydown和keyup事件并渲染小恐龙来作出相应。

而跳跃这个行为是如何实现的呢。

也很简单，就是改变恐龙的y坐标，只不过这里有一个对y坐标变化进行计算的简单公式。

document.addEventListener('keydown',onKeyDown);
document.addEventListener('keyup',onKeyUp);

function onKeyDown(e) {
    if(keycode.JUMP[e.keyCode]) {
        if(!trex.jumping) {
            trex.startJump(6);
        }
    }
}

startJump: function(speed) {
    if (!this.jumping) {
        //切换到jump状态
        this.update(0, Trex.status.JUMPING);
        //设置跳跃速度
        this.jumpVelocity = this.config.INIITAL_JUMP_VELOCITY - (speed / 10);
        this.jumping = true;
        this.reachedMinHeight = false;
    }
}

// 之后在每次GameLoop中更新状态
if (trex.jumping) {
    ctx.clearRect(0, 0, 600, 150);
    trex.updateJump(deltaTime);
}

updateJump: function(deltaTime) {
    //帧切换速率
    var msPerFrame = Trex.animFrames[this.status].msPerFrame;
    //经过的帧数
    var framesElapsed = deltaTime / msPerFrame;
    //更新y轴坐标
    this.yPos += Math.round(this.jumpVelocity * framesElapsed);
    //由于速度受重力影响，需要对速度进行修正
    this.jumpVelocity += this.config.GRAVITY * framesElapsed;

    //达到最小跳跃高度
    if (this.yPos < this.minJumpHeight) {
        this.reachedMinHeight = true;
    }
    //达到最大高度后停止跳跃
    if (this.yPos < this.config.MAX_JUMP_HEIGHT) {
        this.endJump();
    }
    if (this.yPos > this.groundYPos) {
        this.reset();
        this.jumpCount++;
    }
    this.update(deltaTime);
}

2.5 碰撞检测

碰撞检测的概念，经常玩游戏的同学肯定很熟悉，特别是格斗游戏中，碰撞检测应用最为广泛和复杂。

简单来说，碰撞检测就是指两个物体的碰撞（坐标的交错）触发的事件。

这么一说就很清晰了，我们将小恐龙和障碍物的外层套上一个矩形的外壳，通过判断x，y坐标是否有重合来判断是否发生了碰撞，如果碰撞发生，则游戏失败进入判定阶段。

/**
* 碰撞盒子
* @param x    {number} 盒子x坐标
* @param y    {number} 盒子y坐标
* @param w    {number} 盒子宽度
* @param h    {number} 盒子高度
*/
function CollisionBox(x, y, w, h) {
    this.x = x;
    this.y = y;
    this.width = w;
    this.height = h;
}

使用boxCompare方法检测两个盒子是否发生碰撞：

/**
* 碰撞检测
* @param tRexBox {Object} 霸王龙的碰撞盒子
* @param obstacleBox {Object} 障碍物的碰撞盒子
*/
function boxCompare(tRexBox, obstacleBox) {
    var tRexBoxX = tRexBox.x,
        tRexBoxY = tRexBox.y,
        obstacleBoxX = obstacleBox.x,
        obstacleBoxY = obstacleBox.y;

    return tRexBoxX < obstacleBoxX + obstacleBox.width && tRexBoxX + tRexBox.width > obstacleBoxX && tRexBoxY < obstacleBoxY + obstacleBox.height && tRexBox.height + tRexBoxY > obstacleBoxY;
}
